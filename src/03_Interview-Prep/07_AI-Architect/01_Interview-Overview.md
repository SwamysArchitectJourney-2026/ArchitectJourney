# AI + .NET Architect Interview Overview

**Interview Focus**: Azure AI + .NET Architect with Kubernetes positions require deep expertise in:

- Cloud-native .NET microservices architecture
- Azure AI services (OpenAI, Machine Learning, Cognitive Services)
- Kubernetes (AKS) for container orchestration
- Scalable, secure, enterprise-grade solutions

---

- Sets up for technical deep-dives

---

## Interview Structure Expectations

**Typical interview flow:**

1. **Self-introduction** (2 minutes)
2. **Architecture whiteboarding** (20-30 minutes)
   - Draw scalable enterprise solutions
   - Explain trade-offs and design decisions
   - Discuss scalability, security, observability
3. **Technical questions** (15-20 minutes)
   - Azure AI service selection
   - AKS configuration and best practices
   - .NET microservices patterns
4. **Behavioral questions** (10-15 minutes)
   - STAR method responses
   - Leadership and problem-solving examples

---

## Key Areas to Master

**Priority order for preparation:**

1. **Azure AI landscape** - When to use Azure OpenAI vs Azure Machine Learning vs Cognitive Services
2. **AKS fundamentals** - GPU node pools, scheduling, autoscaling
3. **Model serving & MLOps** - KServe, Seldon, deployment patterns
4. **Scaling strategies** - HPA, KEDA, cluster autoscaler
5. **Security & governance** - Managed identity, Key Vault, Private Link
6. **Observability** - App Insights, Prometheus, Grafana
7. **Cost optimization** - GPU vs CPU, spot instances, right-sizing

---

## What Interviewers Look For

**Technical depth:**

- Can you design scalable architectures?
- Do you understand Azure AI service trade-offs?
- Can you configure AKS for AI workloads?

**Communication:**

- Can you explain complex systems clearly?
- Do you ask clarifying questions?
- Can you whiteboard effectively?

**Problem-solving:**

- Can you handle constraints (cost, latency, security)?
- Do you consider non-functional requirements?
- Can you justify design decisions?
